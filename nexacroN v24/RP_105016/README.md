- Grid 표현 기준 마지막 row 클릭시 다음 row로 자동으로 이동하는 기능
- Grid에서 화면 출력 기준 마지막row값을 알수가 없음
- 고객은 e.canvasy가지고 와서 처리를 했는데 grid의 summary나 Grid의 사이즈가 다를 경우 문제가 발생함
- 우선 가이드로 진행할려고 샘플을 만들어서 전달함
- 추후 신규로 전환 될수가 있어서 아래의 내용으로 답변을 남김

* 현황 및 배경
  해당 요구사항은 신규 기능 요청에 해당합니다.
  Grid 화면상 마지막 Row를 확인할 수 있는 기능은 현재 제품에서 제공하지 않습니다.
  유사한 접수 사례도 없어 마이너한 기능으로 판단되어 가이드로 전달합니다
* 기능 구현 방식 (샘플 가이드)
  아래와 같은 방식으로 직접 구현하여 사용하실 수 있습니다.
  1. Grid 전체 높이를 가져옵니다.
  2. Head 및 Summary 영역의 높이를 제외합니다.
  3. 남은 높이를 Row 높이로 나누어 화면상 표시되는 Row 수를 계산합니다.
  4. 계산된 값을 기준으로 클릭한 row가 화면 출력상 마지막 row일 경우 row_position을 이동하는 방식으로 처리합니다
     ※ 마지막 Cell이 겹치거나 소수점이 발생하는 경우, 상황에 맞게 보정하여 사용하시기 바랍니다.

* 샘플 가이드로 해소되지 않을 경우 → 신규 기능 요청 절차
  가이드로 해결이 어렵다고 판단되시면 신규 기능으로 요청 해주시며, 아래 3가지 사항을 상세히 작성해 주시기 바랍니다.
  1. 정확한 사용 케이스 : 고객사에서 해당 기능이 필요한 구체적인 상황 및 시나리오
  2. 기능 요청 날짜 정확하게 파악
     내부적으로 기능 추가가 가능하더라도 고객사가 원하는 날짜까지 불가능 하면 조율이 필요함
  3. 마지막 Row 동작 방식 명확 화
     화면에 일부만 표시(잘린) Row를 클릭할 때 동작하는 것인지
     화면에 완전히 표시된 마지막 Row를 클릭할 때 동작하는 것인지
     (참고 파일: RP_105016_np_Grid_cellclick_rowposition(1).xfdl → 3번째 Grid Body 마지막 Row 확인)
  4. 역방향 이동 여부 : 마지막 Row 클릭 시 다음 Row로 이동하는 기능만 필요한지, 역방향(화면 출력서 첫번째 Row 클릭시 이전 row로 이동)도 함께 고려해야 하는지
