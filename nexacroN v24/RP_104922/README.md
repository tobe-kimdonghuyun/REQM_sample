  - 본 문의는 D3 차트를 Nexacro의 WebBrowser/WebView 컴포넌트에 HTML로 연결하는 방식이 아닌, D3 API를 직접 Nexacro 컴포넌트 또는 스크립트 영역에서 활용하는 방식의 가능 여부에 관한 내용입니다.  

   - D3 차트의 API는 Nexacro 스크립트 영역에서 어느 정도 사용이 가능합니다. 다만, D3 차트가 화면의 특정 영역에 렌더링되기 위해서는 해당 영역의 ID를 참조해야 하는데, 이 과정에서 엔진 내부 변수를 직접 참조해야 하는 구조적인 한계가 있습니다.
    
     (var containerId = this.Div00._unique_id + ".form:container";) 

   - 위와 같이 엔진 내부 변수를 직접 참조하는 방식은 현재 공식적으로 지원하지 않고 있습니다.

   - 따라서 D3 차트를 Nexacro 환경에서 구현하고자 하는 경우, Module Developer를 통해 커스텀 컴포넌트로 개발하는 방식을 통해 구현하시기를 권장합니다.
   - 유선 상으로 관련 가이드 샘플 제공에 대해 안내드린 바 있으나, 해당 샘플은 공식 가이드로 배포하기 어려운 사정이 있어 현재로서는 공식적인 샘플 제공이 어려운 점 양해 부탁드립니다. 

   샘플을 보면 generate된 이후 div를 접근하기 위하여 divObj._unique_id을 가져와서 사용한다. 
   해당 방식은 외부 가이드 코드로 전달사 내부변수 사용을 통한 가이드를 제공하지 않기 때문에 샘플을 전달하지 않았음 
   과거 rmatechart업체 쪽에서 넥사크로 webbrowser 형태로만 제공하다가 속도의 이슈로 인해 커스텀컴포넌트로 개발하여 제공한 이력이 있음 
   하자민 커스텀컴포넌트도 넥사크로 내부변수 사용했기 때문에 정상적인  커스텀은 아님
